# GCN Model Architecture Optimization on Cloud Run (GPU)

This example shows how to use AlphaEvolve to optimize the architecture of a Graph Convolutional Network (GCN) for molecular property prediction, utilizing a remote evaluator running on **Cloud Run with GPUs**.

The target task is predicting molecular solubility using the `adme_sol` dataset (regression) from the [Survey-and-Benchmarks-of-DL-for-Molecular-Property-Prediction](https://github.com/Zongru-Li/Survey-and-Benchmarks-of-DL-for-Molecular-Property-Prediction-in-the-Foundation-Model-Era.git) repository.

---

## Architecture

This example uses **Remote Evaluation** to offload heavy model training to the cloud while keeping the evolution loop control local:

1.  **Orchestrator (`gcn_optimize_remote.py`)**: Runs locally. It communicates with the AlphaEvolve API to get candidate code mutations (generated by Gemini), sends them to the Cloud Run evaluator, and submits the resulting scores back to AlphaEvolve.
2.  **Evaluator Service (`evaluator_service.py`)**: Runs on Cloud Run. It receives the candidate code, writes it to the benchmark directory, trains the model for 100 epochs on an **NVIDIA L4 GPU**, parses the PearsonR metric, and returns it.
3.  **Self-Contained Container**: The evaluator's `Dockerfile` automatically clones the benchmark repository and installs all dependencies (PyTorch, PyG, DGL, CUDA) at build time.

## Setup Instructions

### 1. Prerequisites

*   Follow the root [Get started](https://docs.cloud.google.com/gemini/enterprise/docs/alphaevolve/developer-guide/get-started) guide to provision AlphaEvolve in your GCP project and get your **Gemini Enterprise App ID** (`GE_APP_ID`).
*   Ensure the **Artifact Registry** and **Cloud Build** APIs are enabled in your project.
*   Authenticed `gcloud` CLI:
    ```bash
    gcloud auth login
    gcloud auth application-default login
    ```

### 2. Configure Environment

Create a `.env` file in this directory (you can copy it from `example.env` if you create one, or use the template below):

```env
PROJECT_ID=your-gcp-project-id
GE_APP_ID=your-alpha-evolve-engine-id
LOCATION=global
# EVALUATOR_URL will be set after deploying to Cloud Run
EVALUATOR_URL=https://...
```

### 3. Build and Deploy the Evaluator to Cloud Run

We deploy the evaluator to Cloud Run with 1 L4 GPU.

1.  **Configure Docker to authenticate with Artifact Registry**:
    ```bash
    gcloud auth configure-docker us-central1-docker.pkg.dev
    ```

2.  **Submit the build to Cloud Build**:
    ```bash
    gcloud builds submit --tag us-central1-docker.pkg.dev/YOUR_PROJECT_ID/molecular-predict/gcn-evaluator:latest .
    ```

3.  **Deploy to Cloud Run with GPU**:
    ```bash
    gcloud beta run deploy gcn-evaluator \
        --image=us-central1-docker.pkg.dev/YOUR_PROJECT_ID/molecular-predict/gcn-evaluator:latest \
        --gpu=1 --gpu-type=nvidia-l4 \
        --no-cpu-throttling \
        --no-gpu-zonal-redundancy \
        --cpu=4 --memory=16Gi \
        --concurrency=1 \
        --allow-unauthenticated \
        --region=us-central1
    ```
    *Note: We use `--allow-unauthenticated` for simplicity in this demo. For production, secure it and configure OIDC authentication in the client.*

4.  **Update `.env`**: Copy the `Service URL` from the deployment output and set it as `EVALUATOR_URL` in your `.env` file (append `/evaluate` to the URL).
    *   Example: `EVALUATOR_URL=https://gcn-evaluator-xxxxxx.a.run.app/evaluate`

## Run the Experiment

1.  **Install dependencies**:
    Make sure you have installed the `alpha_evolve` package in your virtual environment:
    ```bash
    pip install -e "../..[examples]"
    ```

2.  **Run the evolution loop**:
    ```bash
    python gcn_optimize_remote.py
    ```

The orchestrator will:
*   Register the experiment with AlphaEvolve.
*   Submit the seed program (`gcn_original.py`) as the initial candidate.
*   Run the controller loop to evaluate 20 candidates (concurrency=2).
*   Print the top-performing architectures and their PearsonR scores at the end.

## Cost Estimate

For a standard run of 20 candidates and 100 epochs per candidate (using Gemini 3.5 Flash for mutations and 1x NVIDIA L4 GPU on Cloud Run for evaluation):

*   **Cloud Run GPU Compute**: ~$1.00 (active request billing for L4 GPU, vCPU, and Memory)
*   **AlphaEvolve API (Inference + Agent)**: ~$1.00 (estimated token usage using Gemini 3.5 Flash at AlphaEvolve rates: $4.50/1M input, $27.00/1M output)
*   **Total Estimated Cost**: **~$2.00**
